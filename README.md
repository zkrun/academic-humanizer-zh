# Academic Humanizer ZH · 中文学术论文写作去 AI 感

> 检测中文学术论文文本的「AI 痕迹风险率」，定位到段落和句子，再在**不改变事实、数据、引用**的前提下改写降低它，并给出**降前 / 降后**的对比。

一个 [Claude Code](https://claude.com/claude-code) Skill，外加一个**可独立运行**的 Python 打分器。
面向论文、综述、报告等中文学术文本。它会指出具体哪一段、哪一句有问题，对应到哪条痕迹、扣了多少分，改完再跑一遍看分数降了多少。

## 它解决什么问题

AI 生成的中文学术文本有一批稳定的「腔调」：模板开头（*随着……不断发展*）、工整排比（*首先……其次……再次*）、
空洞结尾（*具有重要意义，前景广阔*）、空泛归因（*研究表明*却无出处）、高频润色词（*赋能、多维协同、高质量发展*）、
以及把没测过的数字写成*验证了 / 充分证明了*。

这些痕迹**可以被规则识别**，也**可以被有针对性地改写掉**——而不是靠加错别字、口语化来「伪装」。
本项目把这件事做成一个闭环工具。

---

## 核心特点

- 🎯 **可解释的 AI 率**：每一分都能追溯到一条具体痕迹（S1–S7 / C1–C3），不是黑箱数字。
- 🔁 **检测 → 降 AI → 重测 闭环**：改写后自动重新打分，给出「降前 52% → 降后 0%」式对比。
- 📄 **段落 / 句子级定位**：告诉你哪一段、哪一句风险最高，以及为什么。
- 🧮 **混合引擎**：能正则化的表层痕迹交给 Python 脚本确定性打分，跑整篇快、不花 token；需要语言理解的语义层交给 Claude。
- 🛡️ **事实保护**：默认不改数据、引用、方法、结论，不臆造实验 / 样本 / p 值 / 文献。
- 📦 **支持 `.txt / .md / .docx`**，自动跳过参考文献、代码围栏、目录、表格、公式、图表标题、各级标题。
- ⚙️ **脚本可独立使用**：不依赖 Claude 也能跑出一份 AI 痕迹报告。

## 工作原理

### 三步闭环

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  ① 检测      │ ──▶ │  ② 降 AI     │ ──▶ │  ③ 重测      │
│  分段 + 打分  │     │  针对性改写   │     │  重新打分     │
│  逐段 AI 率   │     │  保护事实数据  │     │  降前/降后对比 │
└─────────────┘     └─────────────┘     └─────────────┘
```

1. **检测**：分段，跳过非正文块，对每个正文段落打 AI 率（0–100），列出触发的痕迹与最该改的句子。
2. **降 AI**：优先改写中高 / 高风险段，用「从announcement到reasoning、拆机械排比、空泛价值→具体边界」等方法，保住数据 / 引用 / 方法。
3. **重测**：对改写稿重新跑打分，输出文档级与逐段的降前 / 降后对比。

### 分数怎么来：混合引擎

每段 AI 率 = **表层分（S1–S7，脚本算）** + **语义分（C1–C3，Claude 判）**，封顶 100。

| 层 | 谁来判 | 判什么 | 特点 |
|---|---|---|---|
| 表层 S1–S7 | `scripts/score.py`（正则 / 词表） | 模板开头、排比、空洞结尾、空泛归因、高频词、不安全数据语、升华对比框 | 确定性、可复现、能跑整篇、零 token |
| 语义 C1–C3 | Claude（依据 `rubric.md`） | 节奏过平滑、推理扁平、学科口吻错位 | 需要语言理解，规则做不到 |

> **重要：这个「AI 率」是自研、可解释的痕迹风险分，不是知网 / 维普 / 格子达 / GPTZero 的数字，也不承诺与之相等。**
> 商用检测器是各自的私有模型、会变、无公开 API，任何基于规则的工具都无法精确复现它们的读数。
> 这个分数不用太当真。它的用处是：让分数升高的每条痕迹，正好是改写要消除的东西——检测和改写针对的是同一批目标。

## 评分规则

完整规范见 [`reference/rubric.md`](reference/rubric.md)。下面是权重总表。

### 表层项（脚本确定性打分）

| 代码 | 痕迹 | 例子 | 计分 |
|---|---|---|---|
| **S1** | 模板开头 | 随着…发展 / 在…背景下 / 基于…理论 / 本文围绕… | 单次 +22 |
| **S2** | 排比骨架 | 首先…其次 / 一是…二是 / 理论上…实践上 / 第一…第二…第三 | 单次 +20 |
| **S3** | 空洞结尾 / 综述套话 | 具有重要意义 / 前景广阔 / 提供了新思路 / 综上所述 / 取得丰富成果但仍存在不足 | 每个不同短语 +18，封顶 60 |
| **S4** | 空泛归因 | 研究表明 / 专家认为 / 国内外学者…（且附近无引用标记） | 每处 +13，封顶 39 |
| **S5** | 高频词聚集 | 深入探讨 / 赋能 / 多维协同 / 推动高质量发展（同段 ≥2 个） | ≥2 个 +12，每多 1 个 +5，封顶 36 |
| **S6** | 不安全数据语 | 强断言（验证了 / 证明了 / 显著提高）+ 精确数字 + 全段无来源标记 | 段级 12 + 每个数字 +8，封顶 44 |
| **S7** | 升华对比框 | 不是…而是… / 不仅…而且 / 不在…而在于 / 与其说…不如说（先否定再升华的固定框） | 每处 +8，封顶 20（软信号） |

> S4 有**抑制**：附近 ±30 字出现 `[1]`、`（作者,2020）`、`等[3]` 等引用标记，则视为有据，不计分。
> S6 有**抑制**：全段出现 `表X / 图X / p<0.05 / N= / 显著性检验` 等来源标记，则视为正常结果，不计分。
> S7 是**软信号**：单独一处只到「低」，偶尔的「不是…而是…」是正常表达，与模板 / 排比叠加或成簇时才显著。

### 语义项（Claude 依据 rubric 判，并写明理由）

| 代码 | 痕迹 | 计分 |
|---|---|---|
| **C1** | 节奏过平滑（句句长而均衡、每段都工整收尾、无停顿转折） | 0–20 |
| **C2** | 推理扁平（无例外 / 对比 / 边界 / 作者判断，像教科书摘要） | 0–15 |
| **C3** | 口吻错位（学科特有口吻被磨成通用学术腔） | 0–10 |

### 聚合与分级

- **文档 AI 率** = 各正文段 AI 率按**字数加权平均**（跳过的非正文块不计）
- **高风险段占比** = AI 率 ≥ 51 的段数 / 参与打分的段数
- **四档**：`0–25 低` ｜ `26–50 中` ｜ `51–75 中高` ｜ `76–100 高`

## 安装

打分器只依赖 Python，处理 `.docx` 时需要 `python-docx`：

```bash
python --version            # 需要 Python 3.8+（在 3.13 上测试）
pip install python-docx     # 仅在需要读 .docx 时必须
```

作为 Claude Code Skill 使用时，把整个 `academic-humanizer-zh/` 目录放到你的 skills 目录下
（个人级 `~/.claude/skills/`，或项目级 `.claude/skills/`），Claude Code 会按 `SKILL.md` 的描述自动识别。

## 用法

### 方式一：作为 Claude Code Skill（推荐）

直接用自然语言把文档交给它，例如：

> 帮我检测这篇论文的 AI 率，然后降一降：`D:\paper\thesis.docx`

Skill 会自动完成：跑打分脚本 → 补语义分 → 出**检测报告** → 改写高风险段（导出新文件）→ 重测 → 出**降 AI 报告**。

粘贴**单段**时，它会直接内联判分、给改写、报新风险，不必跑脚本。

### 方式二：独立运行打分器（不需要 Claude）

```bash
# 打印可读摘要
python scripts/score.py 论文.docx

# 同时写出完整 JSON（供二次处理 / 自己改写后重测）
python scripts/score.py 论文.docx --json 结果.json

# 仅写 JSON 不打印
python scripts/score.py 论文.md --json 结果.json --quiet

# 改写后做降前/降后对比，并把逐段对照写成报告文件
python scripts/score.py 原文.docx   --json 降前.json --quiet
python scripts/score.py 改写稿.docx --compare 降前.json --report 对比报告.md
```

可选参数：

| 参数 | 说明 | 默认 |
|---|---|---|
| `--json <路径>` | 写出完整 JSON 结果 | 不写 |
| `--compare <降前.json>` | 打印降前/降后对比表（按正文段落顺序对齐） | 不对比 |
| `--report <报告.md>` | 与 `--compare` 配合，把降前/降后逐段对照写成 Markdown 报告文件 | 不生成 |
| `--min-len <N>` | 低于该字数的段落不打分 | 15 |
| `--top <N>` | 摘要里展示的高风险段数量 | 8 |
| `--quiet` | 不打印摘要（仅写 JSON） | 关 |

摘要示例：

```
============================================================
文档表层 AI率：52.0%  [中高]   (语义项由 Claude 在 skill 流程中补判)
高风险段占比：66.7%   参与打分 6 段 / 跳过 8 段 / 共 14 块
跳过明细：标题6、参考文献2
------------------------------------------------------------
风险最高的 5 段：
  [块5]  82% 高  «S2 S3 S4»
        国内外学者围绕智能辅导系统开展了大量研究，主要集中在系统设计、应用效果和影响机制三个…
  ...
============================================================
```

## 示例：降前 vs 降后

样例文档 `samples/`（一篇虚构的「智能辅导系统」研究节选）跑通的真实结果：

**文档 AI 率：降前 `52.0%`（中高）→ 降后 `0.0%`（表层）**，研究事实、数字、引用全部保留。

**文献综述段（82% 高 → 改后）**

```
降前：国内外学者围绕智能辅导系统开展了大量研究，主要集中在系统设计、应用效果和影响机制
      三个方面，取得了丰富成果，但仍存在不足。首先，已有研究多关注技术实现；其次，对学
      习者个体差异关注不够；再次，长期效果缺乏追踪。

降后：关于智能辅导系统，已有的讨论大多围绕系统怎么设计、用起来效果如何展开，但有一个分歧
      一直没解决：系统的作用究竟来自技术本身，还是来自它如何适应不同学生。本文更关心后一
      种解释，因为前一类工作虽然不少，却较少追踪同一批学生在较长时间里的变化。
```
*处理：拆掉「国内外学者…三个方面…取得丰富成果但仍存在不足」综述套话和「首先/其次/再次」排比，按未解决的分歧重新组织。*

**结论段（60% 中高 → 改后）**

```
降前：综上所述，本文研究具有重要的理论意义和实践价值，为相关领域提供了新思路，前景广阔。

降后：本文的主要工作，是在一个真实班级里把"系统使用"和"学习过程"分开观察，说明个别化反馈
      在其中可能起到的作用。这一判断受限于单一班级的样本，能否推广到其他学段或学科，还要
      看反馈机制和学生基础是否相近。
```
*处理：去掉「综上所述 / 具有重要意义 / 提供新思路 / 前景广阔」空洞价值，换成具体工作 + 适用边界。*

**结果段（不安全数据）的处理原则**——保留数字，但把「充分证明 / 验证了」降级为「说明 / 有所提高」，并补上「需统一测试集后再判断，暂不作为因果结论」的边界，而**不是**删掉数据或臆造统计检验。

## 适用范围

**文本类型**

- 学位论文、课程论文、期刊论文、会议论文
- 文献综述、研究报告、开题 / 结题材料
- 摘要、引言、方法、结果、讨论、结论、答辩文案

**学科**（按学科调整改写口吻，不一刀切）

- 人文 / 理论：保留概念辨析，不强加「结果表明」式实证腔
- 社科 / 教育：保留样本与方法，谨慎使用因果语
- 医学 / 健康：不改临床声明、剂量、统计结果
- 工程 / 计算机：保留参数、数据集、指标，区分「设计设定」与「实测性能」
- 政策 / 管理：保留制度口吻，去口号

**文件格式**：`.txt` / `.md` / `.docx`（旧版 `.doc` 请先在 Word 里另存为 `.docx`；`.pdf` 暂建议先转文本）。脚本遇到 `.doc`/`.pdf` 等二进制格式会**直接报错提示**，不会硬读成乱码给出错误分数。

**自动跳过、不打分的内容**：参考文献区、代码围栏、目录、表格、公式、图表标题、各级标题、过短段落。

## 局限与诚实声明

- **不等于商用检测器读数**：本工具给的是自研可解释痕迹分，不与知网 / GPTZero 等的数字挂钩；若你的场景以某个真实检测器为准，应以「改写质量」为目标，本分数作过程参考。
- **表层项是启发式正则，会有误报**：例如正当的「第一 / 第二 / 第三」列举、统计语境里的「显著性检验」。脚本已加常见抑制条件，最终需要语义层（Claude）裁量。
- **脚本只算表层（S1–S7）**：节奏 / 推理 / 口吻（C1–C3）需要 Claude 在流程里补判，所以纯脚本给出的 0% 是「无表层痕迹」，不等于综合 0 风险。
- **权重为第一版经验值**：建议拿你自己的真实样本校准后微调（改 `reference/rubric.md` 并同步 `scripts/score.py` 顶部词表）。
- **不为「骗过检测」服务**：目标是让文字回到「真实作者在处理真实问题」的状态，保留事实、数据、引用与论证逻辑；不靠加错误或堆反 AI 免责句来伪装。
