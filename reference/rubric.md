# AI 痕迹评分规范 (rubric)

这是 `academic-humanizer-zh` 的**评分唯一规范来源**。`scripts/score.py` 实现其中的
**表层项（S1–S7）**，Claude 在 skill 流程里补判**语义项（C1–C3）**。两边都以本文件为准，
改权重只改这里、并同步改 `score.py` 顶部的词表。

分数语义：**AI 痕迹风险分**，不是任何商用检测器（知网/维普/格子达/GPTZero）的数字，
也不承诺与之相等。它是可解释的风险指标——每一分都能追溯到下面某一条具体痕迹。

---

## 一、表层项（脚本确定性打分）

逐段扫描正文段落。命中即按下表加分，每类有**单段封顶**，避免一类痕迹刷爆整段。

| 代码 | 痕迹 | 触发条件 | 计分 |
|---|---|---|---|
| S1 | 模板开头 | 段首命中模板句式（随着…发展 / 在…背景下 / 基于…理论 / 针对…问题 / 为…而… / 本文围绕…） | 单次 +22 |
| S2 | 排比骨架 | 出现成套序号词（首先…其次 / 一是…二是 / 理论上…实践上 / 宏观…中观 / 第一…第二…第三） | 单次 +20 |
| S3 | 空洞结尾 | 出现空泛价值语 / 综述套话（具有重要意义 / 前景广阔 / 提供了新思路 / 综上所述 / 取得丰富成果但仍存在不足 …） | 每个不同短语 +18，封顶 60 |
| S4 | 空泛归因 | 出现"研究表明 / 专家认为 / 国内外学者 …"**且** ±30 字内无引用标记 | 每处 +13，封顶 39 |
| S5 | 高频词聚集 | 同段出现 ≥2 个 AI 高频润色词（深入探讨 / 赋能 / 多维协同 / 推动高质量发展 …） | ≥2 个 +12，每多 1 个 +5，封顶 36 |
| S6 | 不安全数据语 | 段内有强断言词（验证了 / 证明了 / 显著+效果词）+ 出现精确数字、**且**全段无来源标记（表X/图X/p</N=/检验） | 段级 12 + 每个数字 +8，封顶 44 |
| S7 | 升华对比框 | 「不是…而是… / 不仅…而且 / 不止…而是 / 不在…而在于 / 与其说…不如说」这类先否定再升华的固定框 | 每处 +8，封顶 20（软信号） |

> 强信号（S3 空洞结尾、S4 空泛归因）按出现次数**累加**——一段堆叠多句套话，本身就是更强的 AI 信号。
> S6 改为**段级**判断：一段结果文字若用精确数字「充分证明/验证了」却没有任何统计来源标记，是最危险的不安全数据语。
> 反之，只要段内出现 `表X / 图X / p<0.05 / N= / 显著性检验` 等来源标记，S6 不计分（有据的结果是正常的）。

### 词表（与 `score.py` 同步）

**S1 模板开头**（段首匹配）：
`随着…(不断/日益/快速)?发展/提升/进步/普及` · `在…背景下` · `基于…(理论/框架/视角/模型)` ·
`依据…视角` · `针对…(问题/现象/需求/挑战)` · `为(了)?…(提高/提升/探究/探讨/研究/解决/实现/满足)` ·
`本文(围绕/针对/基于/旨在/聚焦)`

**S2 排比骨架**（成套出现才算）：
`首先 + 其次` · `一是 + 二是` · `理论上 + 实践上` · `宏观 + 中观` · `第一 + 第二 + 第三`

**S3 空洞结尾 / 综述套话**：
`具有(重要/现实/理论/参考/深远/积极)…(意义/价值/作用)`（需有"重要/现实"等空泛限定词，避免误伤"具有统计学意义"） ·
`前景…广阔` · `提供了?(新的)?(思路/借鉴/参考/方向/启示)` · `开辟…方向` · `综上所述` · `由此可见` ·
`总而言之` · `不言而喻` · `取得了?(丰富/丰硕/大量)?…(成果/进展)` · `但仍?存在…(不足/问题/空白/局限)` · `在前人…基础上`

**S4 空泛归因**：
`专家认为` · `学者(普遍/纷纷)?(认为/指出)` · `研究(表明/显示/发现/证明)` · `业内普遍认为` ·
`有(观点/学者/研究)认为` · `大量研究(表明/证明)` · `国内外(众多/许多)?学者` · `(已有/现有)研究`
　抑制（视为有据，不计分）：±30 字内出现 `[数字]` / `（作者,2020）年份` / `等[数字]` / `见…` 等引用标记。

**S5 高频润色词**：
`深入(探讨/分析/研究)` · `系统(梳理/阐述)` · `综合运用` · `有效(提升/促进)` · `显著(提高/改善)` ·
`充分(说明/体现)` · `不可或缺` · `赋能` · `多维(协同/联动)` · `完善…体系` · `构建…框架` ·
`推动…高质量发展` · `提供…新动能` · `保驾护航` · `重要抓手`

**S6 不安全数据语**（段级判断）：
段内同时满足——强断言词 `验证了` / `证明了` / `充分证明` / `完全证明` / `显著(提高/提升/改善/促进/增强/降低/减少)`；
出现精确数字 `数字%` / `数字.数字` / `数字 次/例/人/组/轮`；
且**全段无**来源标记 `表X` / `图X` / `见表图` / `p</=/>` / `N=` / `显著性检验` / `t/F检验` / `回归系数`。
计分 = 12 + 8×数字个数，封顶 44。

**S7 升华对比框**（软信号）：
`不是…而是…` · `不仅…而且(还/也/更)` · `不[只止]…而是(还/更)` · `不在…而在(于)` · `与其说…不如说` · `与其…不如`
　先否定一个平庸说法、再「升华」到高级说法的固定句式，是中文 AI 腔的稳定特征——连语言模型自己想把话说漂亮时都会反射性地用它。
　但偶尔也是正常表达，故**低权重**：每处 +8、封顶 20。单独一处只到「低」，与模板/排比叠加或成簇时才显著。

## 二、语义项（Claude 判断，仅对脚本标记为中风险以上的段落）

| 代码 | 痕迹 | 判断要点 | 分值 |
|---|---|---|---|
| C1 | 节奏过平滑 | 句句长而均衡、每段都以工整总结收尾、毫无停顿/插入/转折 | 0–20 |
| C2 | 推理扁平 | 没有例外、对比、边界、方法理由或作者判断，像教科书摘要 | 0–15 |
| C3 | 口吻错位 | 学科特有口吻被磨成通用学术腔（如人文被写成"结果表明"式） | 0–10 |

Claude 给语义分时要在报告里写明扣分理由，不能只给数字。

## 三、聚合与分级

- **段落 AI率** = `min(100, S1..S7 之和 + C1..C3 之和)`
- **文档 AI率** = 各正文段落 AI率按**字数加权平均**（被跳过的非正文块不计入）
- **高风险段占比** = (AI率 ≥ 51 的段数) / (参与打分的段数)

四档分级（与 SKILL.md 既有口径一致）：

| 区间 | 等级 |
|---|---|
| 0–25 | 低 |
| 26–50 | 中 |
| 51–75 | 中高 |
| 76–100 | 高 |

`score.py` 把 `surface_score ≥ 26` 的段落标记为 `needs_semantic_review = true`，
提示 Claude 对这些段落补语义分。

## 四、跳过规则（不参与打分的非正文块）

参考文献区（命中"参考文献/References/致谢/附录"标题后，其后全部跳过）、**代码围栏（```/~~~ 之间整块跳过）**、
目录行（含 `…… 页码` 引导点）、表格行、公式行（数学符号密集且自然语言字符极少的短行）、
图表标题（`图X /表X` 开头的短行）、各级标题（短、无句末标点，或 `第X章` / 数字编号 / `#` 开头）、过短段落（默认 < 15 字）。

---

## 五、已知局限（诚实声明）

- 表层项是启发式正则，会有误报（如正当的"第一/第二/第三"列举、`显著性检验`语境里的"显著"）。
  脚本已加常见抑制条件，但仍需 Claude 在语义层做最终裁量。
- 分数是**风险提示**，用于定位问题和驱动改写；不等同任何商用检测器读数。
- 权重为第一版经验值，应拿真实样例校准后微调。
